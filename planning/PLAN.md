# Plan — HSK-5 Mandarin Tutor (QLoRA fine-tune)

> What we're building and the plan for getting there. The "why" and "what"; `ARCHITECTURE.md` is the "how".

## Purpose
Fine-tune a small instruct LLM into a Mandarin tutor that answers at a consistent **HSK-5 level** (scaffolds, doesn't escalate), trained on synthetic data from a larger teacher model, and ship it as a Gradio chat demo. Dual goal: modern fine-tuning/shipping-stack fluency + a genuinely useful study tool.

## Scope
- **Now (this phase):** lightweight "vibe" prototype — small, shippable, not research-rigorous.
- **In scope:** synthetic data gen, one QLoRA run, light before/after eval, a runnable Gradio demo, a short README.
- **Non-goals (explicitly not doing):** big datasets, hyperparameter sweeps, rigorous benchmarks, RLHF/DPO, multi-model comparison, production serving/scaling.

## Key features / capabilities
- Six tutoring task types, HSK-5-grounded: `explain_word`, `use_in_sentence`, `correct_sentence`, `translate`, `example_dialogue`, `grammar_point`.
- **Fully bilingual** responses, held at HSK-5 level (scaffolded, not escalating; above-level content simplified + flagged).
- **Reading layer (app-side, deterministic):** pinyin ruby over every character (`pypinyin`), per-word hover glosses (`jieba` + bundled CC-CEDICT), rendered as HTML `<ruby>` + `title`. Not model output → always-correct pinyin, zero training data.
- Synthetic training set (~800–1,000 pairs) generated by Claude, seeded from a real HSK-5 vocab/grammar list, with a ~10% held-out eval split.
- QLoRA adapter merged into the base model → Gradio chat (with the reading layer) → optional HF Spaces.
- **Roadmap (v2, deferred):**
  - **Curriculum coach** — Claude (planner, has memory) discusses goals + writes `curriculum.md` / `progress.md`; `app.py` injects `curriculum.md` into Qwen's prompt so the tutor drills what you're focused on. App-layer only, no retraining. See DECISIONS 2026-07-10.
  - Text-to-speech (read the tutor's Chinese aloud) and voice chat (Whisper-class ASR input).

## Roadmap (milestones / phases)
> Timeboxed to ~2–3 sessions. These group the tasks in TASKS.md.
- **M1 — Data:** `config.py` + HSK-5 seed + `gen_data.py` → generate & sanity-check `train.jsonl` / `eval.jsonl`.
- **M2 — Train + eval:** `train.py` (QLoRA SFT, Colab) → run → `eval.py` before/after on held-out prompts + rubric.
- **M3 — Ship:** merge adapter → `app.py` Gradio chat → README with before/after examples → optional HF Spaces.

## Success criteria (definition of done)
- Fine-tuned model **noticeably better than the base** at HSK-5 tutoring on my held-out prompts (qualitative + rubric).
- A runnable Gradio demo.
- A short README with before/after examples.

## Constraints
- **Hardware:** Apple Silicon M5 Pro / 24GB (local for data-gen + app); **paid Colab** for the QLoRA training run.
- **Base model:** Qwen2.5-7B-Instruct (chosen for tutor reliability). Local demo serves it **quantized** (GGUF/llama.cpp or MLX); free HF Spaces can't host a 7B.
- **Teacher model:** Claude (Anthropic API) for data generation.
- **Timebox:** ~2–3 sessions. Keep it small.

## Open questions
- HSK-5 seed list: use a curated in-repo subset (~150 words + grammar points) vs. sourcing a full ~1,300-word list — starting with a curated seed; revisit if variety is thin.
- Deploy to HF Spaces or leave as local Gradio only — decide at M3.
- Whether 1.5B quality is enough or we bump to 3B — decide after M2 eval.
- Bilingual output format: free-form (Chinese then English) for now; tighten if the annotator or eval needs a stricter Chinese/English separation.
