Ship · finish locally

Finish on the Mac

The T4 is too memory-cramped to merge a 7B in fp16. Your Mac (24GB) does it easily — and the merge + quantize steps need no GPU and none of the training libraries. Here's the whole local finish.

Why this works: merging = fp16 weight math (fits in 24GB RAM), and quantizing = a file conversion (llama.cpp). Neither needs CUDA or bitsandbytes. The only cost is a one-time ~15GB base-model download from Hugging Face.
Disk: you'll transiently use ~40–50GB (15GB base cache + 15GB merged + a 15GB f16 gguf + the 4.7GB final). Make sure you have the room; you can delete the intermediates after.

Steps

1

Download the adapter from Colab

In a Colab cell, zip and download the trained adapter (~100MB):

import shutil
from google.colab import files
shutil.make_archive('hsk5_adapter', 'zip', 'outputs')
files.download('hsk5_adapter.zip')
2

Unzip it into the project's outputs/

cd ~/Documents/ml_projects/chinese_tutor
mkdir -p outputs
unzip ~/Downloads/hsk5_adapter.zip -d outputs

You should see outputs/adapter_config.json and outputs/adapter_model.safetensors.

3

Install the local deps

# from the project folder, ideally in your .venv
python3 -m pip install -r requirements.txt
python3 -m pip install peft            # merge needs peft; it's not in the local reqs

(bitsandbytes is not needed here — we load in fp16, not 4-bit.)

4

Merge the adapter into the base

python3 merge.py

The first run downloads Qwen2.5-7B from Hugging Face (~15GB, one-time, cached) and writes the merged model to outputs/merged/. On the Mac it loads straight into RAM — no GPU, no offload. Takes a few minutes after the download.

5

Build llama.cpp (for the GGUF conversion)

git clone --depth 1 https://github.com/ggerganov/llama.cpp
cmake -B llama.cpp/build -S llama.cpp
cmake --build llama.cpp/build --config Release -j
python3 -m pip install gguf sentencepiece protobuf

~5 minutes; it builds with Metal support automatically on Apple Silicon.

6

Convert + quantize to the served GGUF

python3 llama.cpp/convert_hf_to_gguf.py outputs/merged \
    --outfile outputs/hsk5-tutor-f16.gguf --outtype f16

./llama.cpp/build/bin/llama-quantize \
    outputs/hsk5-tutor-f16.gguf outputs/hsk5-tutor-q4_k_m.gguf Q4_K_M

Result: outputs/hsk5-tutor-q4_k_m.gguf (~4.7GB) — the file app.py loads. You can now delete outputs/merged/ and the f16 gguf to reclaim disk.

7

Run the demo 🎉

python3 get_cedict.py     # once — hover glosses
python3 app.py            # http://localhost:7860

That's the finish line — your fine-tuned tutor, running locally with the pinyin + hover reading layer.

If merge.py is slow or you hit a snag, paste it to me — but on a 24GB Mac loading 14GB of fp16 weights is comfortable, so it should be smooth. The download is the only real wait.
