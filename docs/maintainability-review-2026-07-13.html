Maintainability & readability review

The whole codebase (~4,150 lines: 8 Python files, the extracted JS/CSS, the flashcards widget, setup.sh) graded against the new CLAUDE.md preference: "clear names, small focused functions, comments only where the code can't speak for itself; simple over clever." Three parallel reviewers; every headline claim spot-verified.

Verdict: the codebase comfortably meets the standard. Names are clear, functions are focused, no dead code was found anywhere, and the comment culture is the best kind — recording whys (Gradio quirks, lock discipline, timing races) the code can't say. The real debt is concentrated in one failure mode: comments that were true when written and became lies during fast iteration — mostly leftovers of the 7B→14B pivot and this week's refactors. Those are exactly the things that mislead the next reader, and they're all cheap to fix.
