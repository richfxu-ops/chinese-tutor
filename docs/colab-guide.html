Running the training on Colab

A first-timer's guide. Google Colab is a free/paid service that runs Jupyter notebooks on a cloud GPU, in your browser. We use it because training needs an NVIDIA GPU your Mac doesn't have.

Before you start, make sure the data exists locally — you already ran gen_data.py, so data/train.jsonl and data/eval.jsonl are ready. You'll upload 6 files total (below).

The 6 files you'll upload

All are in your project folder ~/Documents/ml_projects/chinese_tutor:

  • config.py, train.py, eval.py, merge.py — in the top folder
  • data/train.jsonl, data/eval.jsonl — in the data/ subfolder

Steps

1

Open Colab & load the notebook

Go to colab.research.google.com (sign in with your Google account). In the popup: File → Upload notebook (or the Upload tab), and pick train_colab.ipynb from your project folder. It opens as an interactive notebook — a column of "cells" you run top to bottom.

2

Turn on the GPU

This is the one setting that matters. Runtime → Change runtime type → under Hardware accelerator pick GPU. Since you have paid Colab, choose L4 or A100 if offered (a free T4 also works, just slower). Click Save.

If you skip this, train.py will stop with a clear "No CUDA GPU" message — that's the guard we built in, not a bug.
3

Run a cell

Hover over a cell and click the ▶ play button on its left (or press Shift+Enter). Run the cells in order, top to bottom. A spinner means it's working; a green check means done. If Colab warns "notebook was not authored by Google", click Run anyway.

4

Cells 1–3: check GPU, install, upload files

  • Cell 1 (!nvidia-smi) prints your GPU. If it errors, the GPU isn't on — redo step 2.
  • Cell 2 installs the training libraries (~1–2 min).
  • Cell 3 shows a "Choose Files" button. Click it and select all 6 files at once (⌘-click to multi-select). It prints what it received — confirm you see the 4 .py files and both .jsonl.
4½

Cell 4: the sanity run — pause here

This runs 20 quick training steps to prove everything works. Watch the printed loss — it should trend downward over the steps. First it downloads Qwen (~a few minutes, one time), then trains.

Stop and check with me before continuing. Copy the loss numbers to me. If loss goes down → we're good, run the full train. If it's flat or shows "NaN", that's the assistant-masking template — I'll fix it before you spend an hour training.
5

Cell 5: the full training run

The real fine-tune — ~30–60 minutes. It prints the loss every 10 steps and runs an eval each epoch. Keep the tab open and check on it occasionally (Colab disconnects idle sessions). When it finishes it prints adapter saved.

6

Cell 6: eval (optional but recommended)

Generates base-vs-tuned answers on held-out prompts → eval_report.md. This is the payoff — where you see if the tutor actually improved. Send me the output.

7

Cells 7–8: merge & package

Cell 7 folds the trained adapter into the model. Cell 8 converts it to the small GGUF file your Mac runs (builds llama.cpp, ~a few minutes). No input needed — just run them.

8

Cell 9: download the model

This downloads hsk5-tutor-q4_k_m.gguf (~4.7 GB) to your computer. Move it into your project's outputs/ folder. That's the file app.py loads — you're done on Colab.

Common hiccups

ProblemFix
"No CUDA GPU" errorGPU runtime isn't on — step 2, then re-run from the top.
Session disconnectedColab drops idle sessions. Reconnect (top-right), but you'll need to re-run from cell 1 (uploaded files are lost on disconnect). For a ~1hr run, just don't close the tab.
"You are not subscribed…" / no L4/A100Pick T4 — it works fine for a 7B QLoRA, just slower.
Out-of-memory during trainingRare on L4/A100; on a T4 tell me and I'll lower the batch size in config.py.
Download is slowNormal — it's ~4.7 GB. Let it finish.
Checkpoints — what to send me: (1) the sanity-run loss (before the full run), (2) the full-run final loss, (3) the eval report. I'll interpret each and we fix anything that comes up.
